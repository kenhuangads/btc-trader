# ₿ BTC 波段訊號 — 每日智能推薦 × 復盤 × 迭代優化

**線上儀表板：<https://kenhuangads.github.io/btc-trader/>**（手機優先，可「加入主畫面」當 App 用）

比特幣合約短線波段（7 天內）的一條龍工具：每個交易日收盤後（BTC 日 K 於 UTC 00:00 收盤，台北 08:00）由
GitHub Actions 全自動抓取市場數據 → 多因子引擎產生**做多／做空／觀望**推薦（含分批限價掛單梯、停損停利、
槓桿與倉位建議）→ 以保守規則**模擬追蹤每一筆推薦**做每日復盤 → 優化器根據復盤結果**自動調參迭代**。

> ⚠️ **免責聲明**：本專案為研究與教育用途的訊號模擬器，**非投資建議**。合約槓桿交易可能損失全部本金；
> 回測與模擬績效不代表未來表現。請自行判斷，僅以可承受損失的資金參與。

## 設計目標

| 目標 | 對應機制 |
| --- | --- |
| **大賺小賠** | 單筆風險固定 1~2%；TP1(+1R) 平 30% 並把停損移到保本、TP2(+2R) 再平 30%、餘倉 2.5×ATR 移動停損讓利潤奔跑；結構停損＋ATR 緩衝（寬停損防插針） |
| **高掛單成功率** | 進場永遠是回檔限價掛單梯（不追價），檔位自動吸附到支撐/壓力價位群前緣；每一檔標示**歷史觸價成交機率**；優化器依實際成交率動態調整掛單深度 |
| **紀律強制** | 連續 2 次停損 → 冷卻 1 日；FOMC/CPI/非農前 48h 風險減半、24h 內不開新倉；持倉上限 7 天；槓桿上限 5x 且強平價必須遠於停損 |
| **誠實復盤** | 同一根 K 棒同時觸及進場與停損時以停損計（吃虧算法）、計入手續費與滑價；信心分數 vs 實際勝率的「校準表」公開檢驗系統是否自我膨脹 |

## 訊號引擎（10 個因子）

融合全球頂級交易員方法論（GCR 反身性與擁擠度、訂單流 OI×CVD 驗證、CrypNuevo 影線回補、謝林點整數關卡等）：

1. **日線趨勢** — EMA20/50/100 排列與斜率、Donchian 區間位置
2. **4 小時結構** — 高低點推移、均線結構
3. **動能** — RSI(14)、MACD 柱變化（超買超賣抑制追價）
4. **資金費率** — 180 日分位極值＝人群擁擠的逆向訊號
5. **OI×價格矩陣** — 增倉上漲/縮倉上漲/增倉下跌/縮倉下跌四象限判讀
6. **CVD 主動買賣流** — taker 買賣量 z 分數，價量背離警示（吸收/誘多偵測）
7. **相對量能 RVOL** — 無量突破視為假突破
8. **影線磁吸** — 未回補 50% 的長影線＝進場埋伏區與止盈磁吸目標
9. **關鍵價位群** — 擺動點＋整數關卡＋Donchian＋影線位聚類，貼壓力減分、踩支撐加分、放量穿越確認
10. **軋空/殺多醞釀** — 費率極端＋增倉＋價格拒跌（賣壓被吸收）的擁擠反轉結構

加權綜合為 **-100~+100 分數**與**信心值**；|分數| ≥ 門檻且信心達標才出手，其餘觀望（並列出「還缺什麼條件」）。

## 迭代優化器（全部有界、全程留痕）

- **因子權重**：以「因子方向 vs 未來 3 日報酬」的滾動命中率緩慢調整（0.3~1.6）
- **掛單深度**：近 20 筆成交率 <55% 拉近、>85% 且期望值差則加深（0.7~1.4×）
- **停損緩衝**：虧損中「疑似插針掃損」占比 >30% 自動加寬（0.4~1.1 ATR）
- **出手門檻**：依勝率與出手頻率微調（±18~32）

每次調整都寫入「優化日誌」（何時、改了什麼、為什麼），儀表板「系統」頁全部公開。

## 架構

```
GitHub Actions（每日 00:07 UTC）
  └─ engine/run_daily.py
      ├─ fetchers.py     多來源容錯：K線 Binance→OKX→Bybit；衍生品固定 OKX 優先（CI 美國節點連不上 Binance）
      ├─ factors.py      10 因子 + 綜合分數/信心
      ├─ signalgen.py    紀律閘門（逆勢/冷卻/總經事件/信心門檻）
      ├─ planner.py      掛單梯、停損停利、槓桿倉位、觸價機率
      ├─ review.py       1h K 精確重播在途單、統計、自動復盤筆記
      ├─ optimizer.py    調參 + 觸價機率曲線重估
      └─ backtest.py     首次部署 walk-forward 回測建立基準（零未來資訊）
  → 產出 docs/data/*.json 並 commit → GitHub Pages 靜態儀表板（PWA）
```

- `state/` 模型權重、參數、全部交易紀錄（可審計）；`history/` 逐日累積衍生品數據，倉庫本身就是資料庫
- 前端零框架、零建置（`docs/`），支援深/淺色、離線快取、iOS/Android 加入主畫面

## 本地執行

```bash
pip install -r requirements.txt
python -m engine.run_daily              # 每日更新
python -m engine.run_daily --bootstrap  # 重建回測基準
python -m http.server 8123 -d docs     # http://localhost:8123 預覽
```

## 常見操作

- **手動觸發更新**：GitHub → Actions → daily-update → Run workflow（可勾 bootstrap 重建）
- **調整風險偏好**：改 `state/model_state.json` 的 `risk_pct_base`（或等優化器自調）
- **資料時區**：所有日期以 UTC 日 K 為準；儀表板顯示台北時間
